What do I do right now?

If you only want the shortest path, use one of these:

The simplest way to work

1. Pick a project
Open All Projects and choose the project you want.
2. Work in DEV
Make the change in the DEV copy first. That is the safe place to work.
3. Save what happened
Write the Last Action and Next Action so the next time is easier.
4. Move it forward
Use the helper buttons to go from local → DEV → LIVE, and then save to GitHub too.

Quick Help

A built-in cheat sheet for the Working State panel so you do not have to remember the rules.
Use This Flow
  • Do work in DEV.
  • Record what just happened in Last Action.
  • Record the next real step in Next Action.
  • Set Promotion State honestly.
  • Save the project before moving on.
Best Default Values
  • Working Truth: DEV
  • While active: DEV In Progress
  • Waiting on you: Awaiting Approval
  • After push: In Sync
  • When unsure: Unknown
Promotion State Meanings
  • In Sync: DEV and LIVE match
  • DEV In Progress: active work in DEV
  • Awaiting Approval: ready for your review
  • Approved but not promoted: approved, not pushed yet
  • Missing DEV: governance problem
Sync Field Meanings
  • Current: up to date
  • Behind: another surface is newer
  • Unknown: not checked yet
  • Missing: does not exist
  • GitHub Behind: local or DEV work is not mirrored yet

How This Hub Thinks

Every governed project is treated as two surfaces: DEV for active work, LIVE for approved publishing.

Working Truth
DEV is the default editing surface.
Promotion
LIVE updates only after your approval.
Missing DEV
Shown as a governance problem, not healthy.
GitHub
Shown near the top so unfinished mirrors stand out.
